# Q: Why is Engineering Index a cross-community bridge? What does it link?

## Answer

engineering/index.md is the catalog of the wiki's secondary domain (software engineering). Degree 24, all 'references'/EXTRACTED from its [[wikilink]] list. 20 of 24 links go to the new actix-web/actix pages (community 2: HttpServer, extractors, handlers, routing, middleware, errors, databases, testing, websockets, actor-model/address/context/arbiter/sync-arbiter + Actix Web, Actix Framework, Tokio, Serde) — so the index itself was pulled INTO community 2. The remaining 4 links reach back into community 0: Understand-Anything, Tree-sitter, Tree-sitter+LLM Hybrid, Pets-vs-Cattle — the pre-existing SE/code-tooling notes that cluster with the LLM-harness body. So it bridges C2 (Actix Web Knowledge) <-> C0 (LLM Harness & Code Understanding). Unlike log.md (a provenance/chronology bridge across ALL topics), Engineering Index is a TAXONOMIC bridge WITHIN one domain: the actix ingest grew engineering/ from ~2 pages into a dense framework cluster, splitting SE into two sub-bodies (framework deep-dive vs code-understanding tooling) with the index as the seam. The bridge is a direct artifact of this ingest.
