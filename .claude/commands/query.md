CLAUDE.md §3.2 워크플로를 정확히 따라:

1. **인덱스 정찰**: `02.wiki/index.md`를 먼저 읽고 질문과 관련 있어 보이는 페이지 후보를 추린다. 필요하면 `02.wiki/reading/index.md`, `02.wiki/engineering/index.md`, `02.wiki/til/index.md`도 확인.
2. **페이지 정독**: 후보 페이지들을 실제로 읽는다. 부족하면 cross-link를 따라 깊이 들어가고, 1차 근거가 필요하면 `02.wiki/sources/` → 그래도 부족하면 `01.raw/`까지 내려가 원문 확인.
3. **답변 작성**:
   - 한국어로, 기술 용어는 영어 병기
   - 모든 주장에 `[[slug]]` 형식 위키 인용을 붙인다 (출처가 없는 추측은 명시적으로 "위키에 근거 없음 — 일반 지식" 표기)
   - 위키와 일반 지식이 충돌하면 위키를 우선하고, 차이를 짧게 노출
   - 관련 페이지가 아예 없으면 솔직히 "해당 주제 페이지 없음" 보고 + 가장 가까운 인접 페이지를 제시
4. **Archive 제안**: 이번 답변이 단순 조회를 넘어 새로운 합성·비교·분석을 만들었다면, 그 결과를 어디에 어떤 slug로 저장할지(`concepts/`, `engineering/`, `overview.md` 한 줄 등) 사용자에게 제안. 사용자가 동의하면 실제로 페이지 작성 + `index.md` 갱신.
5. **로그 기록**: `02.wiki/log.md`에 한 줄 append:
   ```
   ## [YYYY-MM-DD] query | <질문 요약 한 줄>
   ```

규칙:
- raw/는 읽기만 (수정 금지)
- 페이지 안 읽고 추측으로 답하지 않기 — index만 보고 답하는 것도 금지
- "근거 없음"을 인정하는 게 잘못된 인용보다 낫다
- 답변 끝에 **References** 섹션으로 인용한 위키 페이지 목록 정리
