> 자동 생성 **영어 원본 자막(en-orig)** 을 받아 중복 제거·문단 재구성한 트랜스크립트입니다. (원본은 영어 강연이며 채널이 한국어 자막을 입힌 영상)

## 1. 개요 (Overview)

하네스 엔지니어링(Harness Engineering)은 대형 언어 모델(LLM)을 감싸고(Wrapper), 올바른 컨텍스트와 프로세스를 정의하여 완전한 기능의 '에이전트(Agent)'를 구축하는 설계 및 구현 기술입니다. 이는 과거의 '컨텍스트 엔지니어링'에서 한 단계 진화한 개념으로, 단순한 프롬프트 주입을 넘어 에이전트의 행동을 제어하고 자동화된 워크플로우를 구성하는 것을 목표로 합니다.

- **기본 아키텍처:** `AI 에이전트 = 기본 LLM(추론 엔진) + 하네스 래퍼(컨텍스트 및 프로세스 정의)`
    

## 2. 단일 세션 하네스 아키텍처 (Single-Session Harness)

단일 AI 코딩 어시스턴트 세션 내에서 작동하는 하네스는 크게 세 가지 계층으로 나뉩니다.

1. **기본 모델 (Base LLM):** GPT, Claude 등 순수 추론 모델. (단독으로는 파일 시스템 접근이나 명령어 실행 불가)
    
2. **도구 하네스 (Tool Harness):** Claude Code, Codex 등 모델에 터미널 접근권 및 기본 시스템 프롬프트를 제공하는 상용 래퍼.
    
3. **AI 레이어 (AI Layer):** 개발자가 직접 정의하고 제어하는 **최상위 래퍼**. 하네스 엔지니어링의 핵심 영역입니다.
    

### 2.1 AI 레이어(AI Layer) 핵심 구성 요소

에이전트의 컨텍스트와 프로세스를 제어하기 위해 다음 6가지 요소를 활용합니다.

|**구성 요소**|**설명**|**주요 역할**|
|---|---|---|
|**Global Rules**|전역 규칙 및 컨벤션|에이전트가 준수해야 할 코딩 표준, 제약 조건, 패턴 정의|
|**Skills & MCP**|스킬 및 MCP 서버|에이전트에 특정 작업(예: 웹 검색, DB 접근)을 수행할 수 있는 워크플로우 및 기능 부여|
|**Codebase Search**|코드베이스 검색|LSP(Language Server Protocol) 또는 지식 그래프를 활용한 코드 컨텍스트 파악|
|**Hooks**|훅 (이벤트 트리거)|보안(명령어 실행 전 차단) 및 품질 검증(작업 완료 전 린트/테스트 강제) 로직 적용|
|**Sub-agents**|하위 에이전트|특정 세부 태스크를 위임받아 병렬 또는 순차적으로 처리|
|**Context Docs**|온디맨드 컨텍스트|마크다운, Confluence 문서 등 필요시 주입되는 외부 지식|

## 3. 시스템 진화 프로세스 (Mindset & System Evolution)

하네스 엔지니어링은 단순한 기술적 스킬을 넘어, 에이전트를 다루는 **마인드셋의 전환**을 요구합니다.

- **기존의 한계 (Skill Issue):** 에이전트가 실수를 하면 "다음 버전의 모델(예: GPT-6)이 나올 때까지 기다리자"며 모델 탓을 함.
    
- **하네스 엔지니어링의 접근 (System Evolution):** 모든 실패를 '명시적인 규칙'으로 변환하여 하네스를 개선합니다.
    
    - _사례 1:_ 에이전트가 사내 코딩 컨벤션을 어김 ➡️ `agents.md`에 해당 컨벤션 규칙 추가.
        
    - _사례 2:_ 에이전트가 파괴적인 파일 삭제 명령을 실행함 ➡️ 도구 사용 전 차단하는 '보안 훅(Pre-tool use hook)' 추가.
        

**결론:** 인간 개발자가 피드백 루프(센서, 리뷰 에이전트, 스킬 개선)의 주체가 되어 AI 레이어를 지속적으로 진화시켜야 합니다.

## 4. 다중 세션 오케스트레이션 (Multi-Session Orchestration)

하네스 엔지니어링의 최종 진화 형태는 방대한 작업(예: 대규모 PRD 처리)을 여러 개의 작은 에이전트 세션으로 나누어 **자동화된 워크플로우**로 연결하는 것입니다.

### 4.1 핵심 원칙

- 거대한 태스크를 단일 에이전트에게 맡기지 않습니다. (토큰 비효율 및 모델 과부하 방지)
    
- 역할이 명확히 분리된 세션 간에 아티팩트(예: 마크다운 문서)를 통한 핸드오프(Handoff)를 수행합니다.
    

### 4.2 주요 패턴 및 프레임워크

- **PIV 워크플로우 (Plan - Implement - Validate):**
    
    1. **Plan (계획):** 요구사항을 분석하여 마크다운 형태의 계획서 생성 (세션 1).
        
    2. **Implement (구현):** 계획서를 넘겨받아 실제 코드 작성 (세션 2).
        
    3. **Validate (검증):** 작성된 코드를 테스트하고 리뷰 (세션 3).
        
- **Ralph 루프 (Ralph Loop):** 다중 코딩 에이전트 세션을 자동화하는 하네스 아키텍처.
    
    - 큰 작업을 세부 태스크로 자동 분할합니다.
        
    - 완료 조건(예: `done.ext` 파일 생성)이 충족될 때까지 각 태스크를 해결하기 위해 반복(Loop) 및 병렬 에이전트(보안 검토, 코드 리뷰 등)를 실행합니다.
        
    - 인간의 개입(베이비시팅) 없이 복잡한 소프트웨어 개발 수명 주기(SDLC)를 자동화합니다.
        

## 5. 관련 생태계 및 권장 도구

- **Google Cloud Agent CLI:** 에이전트 구축 지침을 스킬 형태로 제공하며, 로컬 환경(Playground)에서의 테스트와 프로덕션 클라우드 배포(옵저버빌리티 포함)를 단일 명령어로 지원하는 오픈소스 CLI.
    
- **Archon:** 맞춤형 하네스(Ralph Loop 등)와 SDLC 프로세스를 쉽게 구축할 수 있도록 돕는 오픈소스 하네스 빌더.
